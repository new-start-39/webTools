# AI 约束

## 一、回复与沟通

1. 为方便用户确认 AI 已读取本约束，AI 每次回复的第一句必须加上“归雁洛阳边”，后面再继续回复正文。
2. 默认使用简体中文交流。

## 二、需求处理原则

3. 在编码前，先帮助用户整理需求、补全边界条件、明确接口与页面职责。
4. 如果需求还不完整，优先通过提问澄清，不要直接进入实现。
5. 每次实现都以最小可用改动为原则，一步一步推进。
6. 用户没有明确要求的内容，不主动扩展实现范围。
7. 可以提出后续建议，但建议与当前实际改动要分开表达。

## 三、JS 项目编码风格

8. 这是一个 JS 项目。在编码风格上，能用 `async/await` 就用 `async/await`；不到特殊且必须使用的情况，尽量不要使用 `.then`、`.catch` 之类的写法；能用箭头函数时，尽量使用箭头函数。
9. 写代码时优先考虑函数复用，避免重复实现相同或相近逻辑。
10. 复杂逻辑要主动拆分函数，保持单个函数职责清晰、长度可控。

## 四、删除代码时的处理

11. 每当删除代码时，都需要确认与其相互关联的代码、样式、脚本、文案或引用是否也应一并删除，避免残留无效内容。

## 五、设计图转 HTML / Mock 数据约束

12. 特别是当用户提供设计图并要求编写 HTML 时，在 mock 数据场景下，如果存在列表循环，必须使用循环方式生成内容，不能直接将列表数据写死在结构中；同时，相关数据需要与页面结构分开，单独放在 JS 中。

## 六、当前环境信息

13. 当前环境是 Windows Server 2025。
14. 执行命令的环境是 PowerShell。
15. 主要编辑文件时需要注意乱码问题，文本文件编码一律以 UTF-8 为准。
16. 当前环境没有 Python，只有 Node.js。
